Η εργασία αποτελείται από δύο cpp αρχεία, ένα με το κύριο τμήμα του κώδικα και ένα με τις συναρτήσεις της Queue.
Υπάρχει επίσης και ένα header για το Queue, καθώς και το Makefile με κάποιες βασικές λειτουργίες.
Το πρόγραμμα εκτελείται όπως στην εκφώνηση, και αν τα ορίσματα δεν είναι ίδια με αυτά της εκφώνησης, χρησιμοποιείται το directory στο οποίο βρίσκεται το executable.
Το πρόγραμμα δημιουργεί δύο directories, ένα για τα named pipes, και ένα για τα .out files, τα οποία διαγράφονται με την make clean.
Οι workers αποθηκεύονται στο Queue με τα pid τους όταν είναι διαθέσιμοι, δηλαδή το Queue ουσιαστικά κρατάει τα pids.
Τα named pipes έχουν ως ονόματα τα pid του αντίστοιχου worker που ασχολείται με αυτά, ώστε ο κάθε worker να μπορεί να έχει πρόσβαση σε αυτά κάνοντας getpid().
Ο manager ελέγχει αν αυτό το Queue είναι άδειο και αν είναι δημιουργεί καινούργιο worker και named pipe.
Αν το queue δεν είναι άδειο, δίνει εντολή σε έναν worker που θα κανει pop.
Πριν ελέγξει το Queue, σπρώχνει σε αυτό όσους workers είναι σταματημένοι, εφόσον έχει δεχτεί αντίστοιχο μήνυμα απο τον handler του sigchld.
Στην εργασία τα named pipes αναφέρονται ως fifo για συντομία, και έχω εξηγήσει τα περισσότερα βήματα στα οποία χρησιμοποιούνται πολλοί buffers με σχετικά ονόματα.
